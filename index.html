Steps for creating Web Design 1 websites

Create Repository

Follow the steps and shreenshot examples below:

  1. In GitHub account, click + sign in upper right corner
  2. Select “New Repository”
  3. Repository Name = (name it after your project)
  4. Check “Public”
  5. Add ReadMe file
  6. Click “Create Repository”




Clone the repository

Follow the steps and screenshot examples below:

This is creating a copy of what is on your online account to your local computer

  1. Click on your repository
  2. Click green “Code” button in upper right corner
  3. Click “Open with GitHub Desktop”
  4. A “Clone a Repository” window will open
  5. Make sure the local path is saving to the correct folder
  6. Click the “Clone” button
  7. Double check your local computer to make sure the files were copied over




Create Dreamweaver site

Follow the steps and screenshot examples below:

Create a new site in Dreamweaver every time you clone a new repository

  1. In Dreamweaver go to:
    Site - New Site - name the site the same name as the repository
  2. For “Local Site Folder” select the folder where your repository was cloned on your computer
  3. Click the “Choose” button
  4. Save
  5. Go to: File - New - Title (this is the title for the tab of your site)
  6. Click “Create” button




Create first webpage (index.html)

Follow the steps and screenshot examples below:

  1. In Dreamweaver go to:
    File - Save As - index.html
    This is a very important filename for the first page of your site.
  2. Save
  3. Enter code into the of your page
  4. Save page






Commit changes

Follow the steps and screenshot examples below:

In GitHub Desktop you should see the index.html file in your new repository

  1. Click the “Commit to main” button Commits can be thought of as snapshots along the timeline of a Git project


Push changes to online repository

  1. Click “Push origin” to “push it up to the internet”




Create a “Gitpage”

Follow the steps and screenshot examples below:

  1. In your GitHub account (from repository view) click “Settings” (last option in menu tabs along the top)
  2. Click pages (left sidebar menu)
  3. Under the Branch action select “Main” from the dropdown
  4. Click Save
  5. Select “Actions” from the top tabs menu. The workflow status may show “in progress”
  6. Once it is complete, go back to:
    Settings - Pages - a url is generated for your live page
  7. Click “Visit Site” button
